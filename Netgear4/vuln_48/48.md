# Netgear Vulnerability

Vendor:Netgear

Product:R8500、XR300、R7000P、R6400v2

Version:1.0.2.160、1.0.3.78、1.3.3.154、1.0.4.128

Type:Command Execution

Author:Jiaqian Peng

Institution:pengjiaqian@iie.ac.cn



## Vulnerability description

We found an Command Injection vulnerability in Netgear router with firmware which was released recently, allows remote attackers to execute arbitrary OS commands from a crafted request.

**Remote Command Execution**

In `httpd` binary:

In the router's `ap_mode.cgi、wlg_adv.cgi` function, `apmode_gateway` is directly passed by the attacker, so we can control the `apmode_gateway` to attack the OS.

> XR300 in `wlg_adv.cgi`
>
> R7000P in `operation_mode.cgi、ap_mode.cgi、wlg_adv.cgi`
>

As you can see here, the input has not been checked. And then,call the function `acosNvramConfig_set ` to store this input.

<div  align="center"><img src="./images/1.png" style="zoom:80%;" /></div>

In `rc` binary:

In `sub_168D0` function, the initial input will be extracted. Eventually, the initial input will cause command injection.
